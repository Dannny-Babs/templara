This walkthrough takes you from install to rendered output, then shows where editing and PDF export fit.

## 1. Install the render stack

```bash
pnpm add @templara/core @templara/renderer @templara/react-renderer @templara/templates
```

Add the editor and PDF helpers when you need them:

```bash
pnpm add @templara/editor @templara/pdf
```

## 2. Render a starter invoice

The fastest path is render-only: take a starter template, pass sample data, and paint the result with `DocumentPreview`.

```tsx
import { renderDocument } from '@templara/renderer';
import { DocumentPreview } from '@templara/react-renderer';
import { invoiceSampleData, invoiceTemplate } from '@templara/templates';

const result = renderDocument({
  template: invoiceTemplate,
  data: invoiceSampleData,
  mode: 'preview',
});

export function InvoicePreview() {
  return <DocumentPreview document={result} />;
}
```

Expected result: a paginated preview surface rendered from template JSON and sample invoice data. The preview is not the editor canvas. It is the final output path the PDF export also uses.

## What you get back

`renderDocument()` returns:

- `pages` — paginated render tree
- `warnings` — actionable layout and data diagnostics
- `repeatAnalyses` — per-repeat pagination metrics

## 3. Check warnings before export

Render warnings are part of the contract. They tell you about missing bindings, repeat layout issues, overflow, and other output concerns.

```ts
if (result.warnings.length > 0) {
  console.warn('Document rendered with warnings', result.warnings);
}
```

## 4. Embed the editor when users need authoring

`DocumentEditor` is controlled by your app. Your app owns persistence; the editor emits template and sample-data changes.

```tsx
import { useState } from 'react';
import { DocumentEditor } from '@templara/editor';
import { invoiceSampleData, invoiceTemplate } from '@templara/templates';
import type { DocumentTemplate } from '@templara/core';

export function TemplateStudio() {
  const [template, setTemplate] = useState<DocumentTemplate>(() =>
    structuredClone(invoiceTemplate),
  );
  const [data, setData] = useState(() => structuredClone(invoiceSampleData));

  return (
    <DocumentEditor
      value={template}
      data={data}
      documentTitle="Invoice Template"
      documentStatus="draft"
      onChange={setTemplate}
      onDataChange={setData}
    />
  );
}
```

The editor shows authored nodes, handlebars, and one repeat row. Use `renderDocument()` plus `DocumentPreview` for final output.

## 5. Export the rendered preview to PDF

PDF export works from the already-rendered preview page elements. The export package does not re-render or reinterpret the template.

```ts
import {
  buildExportFontCss,
  collectExportDiagnostics,
  exportPreviewToPdf,
} from '@templara/pdf';
import type { RenderDocumentResult } from '@templara/renderer';

export async function exportCurrentPreview(document: RenderDocumentResult) {
  const diagnostics = collectExportDiagnostics(document);
  const blocking = diagnostics.diagnostics.filter((item) => item.severity === 'error');

  if (blocking.length > 0) {
    throw new Error(blocking[0].message);
  }

  const pageElements = Array.from(
    window.document.querySelectorAll<HTMLElement>('[data-templara-page-id]'),
  );

  return exportPreviewToPdf(
    pageElements,
    document.pages.map((page) => ({ width: page.width, height: page.height })),
    { title: 'Invoice', fontCss: buildExportFontCss(document) },
  );
}
```

## Keep the pipeline straight

```txt
Template JSON + Data JSON
  -> renderDocument()
  -> RenderDocumentResult
  -> DocumentPreview
  -> exportPreviewToPdf()
```

The editor sits beside this pipeline. It edits template JSON; it is not the final renderer.
